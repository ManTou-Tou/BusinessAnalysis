# Codex Review 规则（强制）

> 本文件是一条硬性规则。Claude Code 在本项目中的所有产出都必须遵守。

## 核心规则

**无论 Claude Code 做任何方案（plan）或编写/构建代码（build code），都必须先经过我的 Codex review，并确认没有问题之后，才可以被采用、使用或实际执行。**

## 具体要求

1. **方案先行**：在动手写代码之前，Claude Code 必须先给出方案（计划、思路、要改动的文件、预期影响）。
2. **等待 Codex review**：方案和代码都要交给我的 Codex 进行 review。
3. **未通过不执行**：在 Codex review 通过（确认没有问题）之前：
   - 不得把方案当作最终方案；
   - 不得把代码合并、部署或投入使用。
4. **通过后才放行**：只有当 Codex review 明确表示「没有问题」后，对应的方案和代码才算「可以使用、可以做过」。
5. **有改动需重审**：如果方案或代码在 review 之后又有修改，需重新经过 Codex review。

## 一句话总结

> Claude Code 提出方案和写代码 → 先给 Codex review → Codex 说没问题 → 才能用。
